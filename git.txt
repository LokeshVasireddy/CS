SSH (Secure Shell)
SSH is a secure network protocol used to connect to remote computers securely and transfer data in encrypted form.
In Git, SSH is used to authenticate and communicate with remote repositories without entering a username and password every time.
You create an SSH key pair (public + private)

Public key is added to GitHub/GitLab

Private key stays on your computer

Git uses the key to authenticate securely

Reset(rewrites history)
-----
git reset moves the HEAD (and optionally branch pointer, staging area, and working directory) to a specified commit.
Git reset --hard  HEAD~1//

1️⃣ git reset --soft 🟢
git reset --soft HEAD~1
What it affects

✅ Moves HEAD

❌ Does NOT touch staging

❌ Does NOT touch working directory
Undo commit but keep changes staged

2️⃣ git reset --mixed 🟡 (DEFAULT)
git reset --mixed HEAD~1
What it affects

✅ Moves HEAD

✅ Resets staging area

❌ Does NOT touch working directory

3️⃣ git reset --hard 🔴 (DANGEROUS)

What it affects

✅ Moves HEAD

✅ Resets staging

✅ Resets working directory


Revert(preserves history)
------
git revert creates a new commit that undoes the changes of a specific commit without rewriting Git history.
Instead of removing a commit, it adds another commit that reverses it.

git revert <commit-hash>
git revert HEAD~3..HEAD

Reverting a merge commit 

git revert -m 1 <merge-commit-hash>
-m 1 → parent branch to keep


Restore
--------
git restore is used to restore files in the working directory or staging area from a commit, index, or HEAD.
It undoes file changes

It does NOT touch commit history

✅ You messed up a file
✅ You want to unstage changes
✅ You want to discard local edits
❌ You want to undo a commit → use git revert

Undo changes not yet staged:git restore file.txt

git restore --staged file.txt
✔ Removes file from staging
✔ Keeps changes in working directory

Rebase
-----
1️⃣ git rebase

👉 Starts a rebase.

Example:

git rebase main


Before
main:      A --- B --- C

feature:         D --- E

After 
main:      A --- B --- C --- D --- E


2️⃣ git rebase --continue

👉 Used AFTER a rebase has started.

Specifically when:

Git paused because of conflicts

You fixed the conflicts

Now Git needs to continue replaying commits


git rebase main     ← start rebase
        ↓
(conflict happens)
        ↓
Fix files manually
git add .
git rebase --continue   ← resume rebase


| Command                 | What it does                  |
| ----------------------- | ----------------------------- |
| `git rebase`            | Starts moving commits         |
| `git rebase --continue` | Resume after fixing conflicts |
| `git rebase --abort`    | Cancel rebase                 |
| `git rebase --skip`     | Skip problematic commit       |
